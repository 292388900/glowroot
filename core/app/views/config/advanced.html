When enabled, all methods that are woven for trace metric timing are renamed with suffix $glowroot$trace$metric$<trace metric name> and a new method is generated in its place that simply calls the renamed method. The purpose of this is so that stack traces gathered by the profiler can see what trace metric timings are active, and the resulting trace profiles can be filtered in the UI by trace metric names. For example, a trace profile can be filtered to see only the profile of code that is executing jdbc queries. Or, even more useful, filter to see only the profile of code that has no trace metric timings, which helps to discover new bottlenecks that would be worth timing via pointcuts. Changes to this setting will take effect on next JVM restart.
