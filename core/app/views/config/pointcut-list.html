Where to apply the pointcut
The class or interface containing the method(s) where the pointcut will be applied.
The method(s) where the pointcut will be applied. The pointcut will also be applied to all overriding methods (including all implementations of the method if the class above is an interface).
The wildcard * is supported anywhere in the method name. Multiple method names (including multiple method names with the wildcard * in them) can be listed with the | separator between them.
What to capture at the pointcut
Multiple pointcuts can share the same metric name, in which case their metrics are aggregated together.
This is a template which can be populated with values from the method invocation:
  • {{0}}, {{1}}, ... resolve to the method parameters
  • {{this}} resolves to the object that the method is invoked on (except for static methods)
  • {{_}} resolves to the return value (if not void)
  • {{methodName}} resolves to the method name, which can be useful if * or | are used to define the method name above
Nested paths are resolved as well, e.g. {{0.size}} and {{this.class.name}}. If a path resolves to null at any point, then the entire path will resolve to null. Finally, resolved value is rendered into the template using String.valueOf().
If checked, and the pointcut is encountered within one of its own spans, a nested span will be captured. This can sometimes lead to a lot of extra noise, e.g. in a fairly common case where the pointcut matches several overloaded methods that cascade to each other.
Any span that exceeds this threshold will have a stack trace captured and attached to it. Empty means never capture a stack trace for this span.
