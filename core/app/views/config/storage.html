Storage

days
Defines how long aggregates should be retained. Aggregate profiles may expire sooner than this, see Profiles/trace entries capped db below.
days
Defines how long traces should be retained. Profiles and trace entries may expire sooner than this, see Profiles/trace entries capped db below.
Profiles and trace entries can be large, and are stored in a capped database for efficient access and capped disk usage. Once this capped database grows to its maximum size, new profiles and trace entries start overwriting the oldest ones.

{{dataDir}}

The data directory where the H2 data file and the capped database are stored. The default value is the directory where the glowroot.jar resides. This can only be changed on the command line, using -Dglowroot.data.dir=<folder>.
