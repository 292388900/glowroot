Outlier profiling

Once a trace exceeds this amount of time, the outlier profiler captures stack traces periodically. The interval between successive stack trace captures is defined by the property directly below. The stack traces captured for an individual trace are merged into a tree view and provide a profile of the time spent in various portions of the code.
The interval at which the outlier profiler captures stack traces (once a trace exceeds the initial delay directly above).
The maximum amount of time that a given outlier trace will be profiled (once a trace exceeds the initial delay above). This is used to limiting the memory requirement of very long traces that would capture potentially many thousands of stack traces (e.g. large batch or background operations).
